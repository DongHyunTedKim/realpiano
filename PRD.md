# PRD — 피아노 코드 & 건반 시각화 웹 서비스 (MiniXi)

---

## Project Overview

피아노 독학자가 코드 이름을 검색하면, 해당 건반이 반짝이며 소리가 나는 **인터랙티브 피아노 코드 시각화 웹 서비스**입니다.  
악보를 읽지 못하거나 이제 막 피아노를 시작한 초보자가, 스마트폰이나 아이패드를 보면대에 올려두고 즉시 코드를 확인하며 연습할 수 있도록 설계합니다.

---

## Product Vision

> "코드 이름만 알아도, 어디를 눌러야 하는지 바로 보인다."

음악 이론을 몰라도 코드를 눈으로 보고, 귀로 듣고, 손으로 따라 칠 수 있는 가장 직관적인 피아노 코드 레퍼런스 도구를 만든다.

---

## Problem Statement

피아노 독학자들은 "Caug는 어떤 건반이야?"라는 질문에 답을 얻기 위해 유튜브를 검색하거나 복잡한 앱을 뒤지는 데 시간을 낭비한다.  
기존 앱들은 기능이 너무 많아 무겁고, 보면대에서 빠르게 참고하기 어렵다.  
**원하는 코드를 입력하면 즉시 건반이 반짝이고 소리가 나는, 단순하고 빠른 도구가 필요하다.**

## User Scenarios

### 시나리오 1 — 코드 즉시 확인
> 유저가 반주 악보를 보다가 "Caug"라는 생소한 코드를 발견한다.  
> 아이패드를 열고 검색창에 "Caug"를 입력한다.  
> 피아노 건반 UI에서 해당 건반들이 반짝이고, 버튼을 누르면 소리가 난다.  
> 유저는 어느 건반을 눌러야 하는지 바로 파악하고 연주를 이어간다.

### 시나리오 2 — 코드 목록 탐색
> 유저가 "7th 코드가 어떤 게 있지?"라고 궁금해한다.  
> 카테고리 목록에서 "7th"를 선택하면 G7, C7, D7 등의 목록이 나온다.  
> 하나씩 탭하며 건반 위치와 소리를 확인한다.

| F-08 | **100개+ 코드 지원** | Major, minor, 7th, maj7, m7, aug, dim, sus2, sus4 등 주요 코드 전체 12개 루트음 × 코드타입 |
| F-09 | **모바일 반응형 UI** | 아이패드/아이폰 세로/가로 모드 모두 최적화 |

## Non-Functional Requirements

| 항목 | 요구사항 |
|------|---------|
| **성능** | 코드 입력 후 건반 하이라이트까지 **200ms 이내** |
| **반응형** | 모바일(320px) ~ 태블릿(1024px) 전 해상도 지원 |
| **접근성** | 색상 외에도 텍스트(음이름)로 정보 제공 (색맹 고려) |
| **오프라인** | 인터넷 연결 없이도 기본 기능 동작 (정적 웹 앱) |
| **설치 불필요** | 앱스토어 설치 없이 브라우저에서 바로 실행 |
| **소리 지연** | 버튼 클릭 후 소리 재생까지 **100ms 이내** |

## Tech Stack

| 영역 | 기술 |
|------|------|
| **프론트엔드** | HTML5, CSS3, Vanilla JavaScript (프레임워크 없이 심플하게) |
| **소리 재생** | Web Audio API — 합성음 (OscillatorNode 기반, 외부 샘플 파일 없음) |
| **코드 데이터** | 코드 구성음 JSON 파일 (정적 데이터, 서버 불필요) |
| **반응형 CSS** | CSS Grid / Flexbox, CSS 변수 기반 테마 |
| **애니메이션** | CSS Keyframes + JavaScript 트리거 |
| **빌드 도구** | 없음 (단순 정적 파일) |
| **배포** | **GitHub Pages** (무료, 정적 호스팅) |

> **AI Agent 개발 참고:** 외부 API 호출 없이 100% 클라이언트 사이드에서 동작하는 정적 웹 앱으로 구현한다. 서버/백엔드 불필요.

## Success Criteria

> 아래 기준을 **모두** 만족하면 MVP 완료로 간주한다.

- [ ] 코드 이름(예: `Caug`)을 검색하면 해당 건반이 정확히 하이라이트됨
- [ ] 코드 버튼을 클릭하면 해당 코드의 소리가 재생됨
- [ ] 아이패드/아이폰 브라우저에서 오류 없이 정상 작동함
- [ ] 비개발자에게 데모 시 별도 설명 없이도 직관적으로 사용 가능함
- [ ] 100개 이상의 코드(12루트음 × 주요 코드 타입)를 지원함

## MVP Scope

MVP에 포함되는 기능:

1. **피아노 건반 UI** — 화면 상단에 고정, **2옥타브(25건반)**, CSS로 건반 직접 구현
2. **코드 검색창** — 텍스트 입력으로 코드 즉시 검색
3. **코드 목록** — 루트음 / 코드타입별 브라우징 UI
4. **건반 하이라이트** — 선택된 코드의 구성음 색상 + 반짝이는 애니메이션
5. **음 역할 레이블** — Root, 3rd, 5th 표시
6. **소리 재생** — 버튼 클릭 시 **Web Audio API 합성음**으로 코드음 동시 출력
7. **한/영 전환** — 음이름 표기 방식 토글 버튼
8. **모바일 반응형** — 아이패드/아이폰 최적화
9. **GitHub Pages 배포** — 별도 서버 없이 정적 호스팅

## AI Context

> **향후 AI 개발 에이전트가 반드시 알아야 하는 컨텍스트**

### 개발 우선순위
1. **피아노 건반 UI + 하이라이트 애니메이션** (핵심)
2. **코드 데이터 JSON** (12루트음 × 주요 코드타입 구성음 정의)
3. **코드 검색 + 목록 브라우징**
4. **Web Audio API 소리 재생**
5. **한/영 음이름 전환**
6. **모바일 반응형 CSS**

### 중요 제약사항
- **서버 없음**: 100% 정적 클라이언트 웹 앱, GitHub Pages 배포
- **외부 의존성 없음**: Web Audio API 합성음 사용으로 외부 사운드 파일 불필요
- **복잡도 낮게**: 1인 개발 + AI 어시스턴트 조합이므로 단순하고 읽기 쉬운 코드 우선
- **모바일 최우선**: 데스크탑보다 아이패드/아이폰 UI를 먼저 설계할 것
- **건반 범위**: 2옥타브(25건반) — C3~C5 또는 C4~C6 범위 권장

### 사용자 기대사항
- "코드 이름 치면 바로 어디 눌러야 하는지 보이는" 즉각적 피드백
- 보면대에 올려두고 한 손으로 탭 가능한 큰 버튼 UI
- 반짝이는 애니메이션으로 시각적 명확성
- 클릭하면 소리가 나서 귀로도 확인 가능

### 구현 시 주의사항
- 피아노 건반은 CSS로 직접 그릴 것 (이미지 파일 X → 확장성)
- **건반은 2옥타브(25건반)** — 흰 건반 15개, 검은 건반 10개
- 흰 건반과 검은 건반의 하이라이트 색상을 명확히 구분
- 음이름 표기: 영어 기본, 한국어(도레미) 토글 지원
- **Web Audio API는 모바일 Safari에서 사용자 제스처(클릭) 이후에만 소리 재생 가능** → 반드시 클릭 이벤트 내에서 AudioContext 초기화
- Web Audio API 합성음 구현: `OscillatorNode` + `GainNode` 조합, 피아노 음역대 주파수 테이블 사용
- 코드 데이터는 별도 `chords.js` 또는 `chords.json` 파일로 분리하여 관리
- GitHub Pages 배포를 고려해 모든 경로는 상대 경로로 작성
